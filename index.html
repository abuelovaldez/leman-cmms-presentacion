Sistema de Alertas

El sistema
avisa solo

Sin necesidad de revisar manualmente. Las alertas se generan automáticamente cuando se cumplen condiciones críticas.

CRÍTICA
Service Vencido
El service programado ya superó su fecha límite o los km/horas establecidos. Requiere atención inmediata para no comprometer la operación.
ALTA
Licencia Vencida
La licencia de conducir de un operador está vencida. El operador no debería operar ningún vehículo hasta renovarla.
MEDIA
Service Próximo a Vencer
El próximo service está a menos de 30 días o dentro del umbral configurado. Es momento de programar el turno en el taller.
BAJA
Activo sin Operador
Un activo figura como "En Uso" pero no tiene operador asignado. Puede indicar un error de carga o un activo prestado sin registrar.
¿Cómo se generan?
Las alertas se crean automáticamente en la base de datos mediante funciones SQL (triggers) que se ejecutan cada vez que se registra un service, se actualiza un operador o cambia el estado de un activo.
¿Cómo se resuelve?
El administrador o supervisor ingresa al módulo de Alertas, revisa el detalle y hace click en "Marcar como Resuelta". La alerta queda registrada en el historial.
⚡ Tiempo real
Las alertas se sincronizan vía WebSockets. El badge del menú se actualiza automáticamente sin necesidad de recargar la página.
